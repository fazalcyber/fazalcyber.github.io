Git Bash Command Documentation for Project Update

 Working Directory

$ pwd

Command Explanation:
pwd stands for "Print Working Directory". It shows the current location in the file system.

Output:

 

The user is currently in the DELL directory.

Changing to Project Directory

$ cd fazalcyber.github.io/

Command Explanation:
cd means "Change Directory". This command moves to the project folder named fazalcyber.github.io.

Output:

 




Pull Latest Changes from GitHub

$ git pull

Command Explanation:
git pull fetches and merges changes from the remote repository to the local system.

Output:

This means there are no new changes on GitHub that need to be pulled.


Adding Files to Git

$ git add --all

Command Explanation:
git add --all stages all the changes (new files, modified files, deleted files) for the next commit.

Output:


Committing Changes

$ git commit -m "day-3 commit"

Command Explanation:
git commit - saves the staged changes with a message. The -m flag adds a commit message.
Message:
"day-1 commit" describes what this commit is about.

Output:

 

This shows that  new files were added to the project.

Pushing Changes to GitHub

$ git push

Command Explanation:
git push -uploads the local repository changes to the remote GitHub repository.

Output:

The push was successful and changes were uploaded to GitHub.

Summary of Commands

Command

       Purpose

Pwd                

  Shows current directory

cd <directory>

  Move to project folder

git pull

  Fetch and merge latest changes

git add --all

  Stage all changes

git commit -m ""

  Save changes with a commit message
