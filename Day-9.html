Windows Network Commands

.

Why Learn Network Commands?

Mastering these commands allows you to:

· Diagnose internet and LAN issues.

· Configure IP and DNS settings.

· Test network performance and connections.

· Access shared files across the network.

· Monitor active connections.

IP Configuration & DNS

1. ipconfig

Description: Displays basic IP information like IPv4 address, subnet mask, and default gateway.

C:\> ipconfig

Output:

 

2. ipconfig /all

Description: Shows detailed configuration for all network adapters, including MAC address and DNS servers.

C:\> ipconfig /all

Output:

 

3. ipconfig /release

Description: Releases the current IP configuration (used in DHCP environments).

C:\> ipconfig /release

Output:

4. ipconfig /renew

Description: Renews the IP configuration after releasing it.

C:\> ipconfig /renew

Output:

5. ipconfig /flushdns

Description: Clears the DNS cache, useful when facing DNS resolution errors.

C:\> ipconfig /flushdns

Output:

 

6. nslookup [domain]

Description: Resolves the IP address for a domain name using DNS lookup.

C:\> nslookup google.com

Output:

Connectivity Testing

7. ping [IP or domain]

Description: Sends ICMP echo requests to test if a host is reachable.

C:\> ping 8.8.8.8

Output:

8. tracert [IP or domain]

Description: Traces the path of packets to a destination showing each hop.

C:\> tracert google.com

Output:

9. pathping [IP or domain]

Description: Combines ping and tracert, also shows packet loss stats.

C:\> pathping 8.8.8.8

Output:
